# Olimp Bot Payment

Отдельный Telegram-бот для продажи подписки на закрытый канал с контентом.
Платёжная система полностью повторяет основной бот (Olimp_Bot): те же способы
оплаты, те же тексты и блоки, та же админка с A/B-тестами цен, рассылкой и
статистикой. Продуктом является доступ в **один** закрытый канал — после оплаты
бот выдаёт персональную одноразовую пригласительную ссылку.

## Как работает оплата

1. `/start` → приветствие и выбор срока подписки: **1 / 3 / 6 месяцев**.
2. После выбора срока — текст с описанием способов оплаты и кнопки выбора метода
   (идентично основному боту).
3. **Boosty** и **Tribute** — сначала предупредительное окно, затем ссылка.
   Выдача доступа автоматическая (после вступления в канал).
4. **Ручная оплата** — реквизиты (карты РФ 🇷🇺, РБ 🇧🇾, USDT) и контакт менеджера.
5. **Crypto Pay** и **Telegram Stars** — счёт создаётся сразу.
6. После успешной оплаты бот отправляет **одноразовую инвайт-ссылку** в закрытый
   канал (бот должен быть администратором канала).

## Команды

**Пользователь:** `/start`, `/buy`, `/status`, `/cancel`
**Админ:** `/admin`, `/stats`, `/ab_stats`, `/broadcast`, `/export_user <id>`,
`/admin_grant <id> [дней]`

## Стек

- TypeScript + Telegraf
- PostgreSQL + Prisma (адаптер `@prisma/adapter-pg`)
- Redis + BullMQ (сессии и очередь рассылки)
- Express (webhook Crypto Pay)

## Быстрый старт

```bash
cp .env.example .env      # заполните токены и PAID_CHANNEL_ID
npm install
npm run db:generate
npm run db:migrate:deploy # применить миграции к БД
npm run build
npm start
```

Или через Docker (на старом Docker Compose v1 — команда `docker-compose` через
дефис; на новом v2 — `docker compose`):

```bash
cp .env.example .env
docker-compose -p olimp_bot_payment up -d --build
```
